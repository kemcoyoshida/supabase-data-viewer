# HTTPSサーバーの起動方法

## 簡単な起動方法

### 方法1: バッチファイルを使用（推奨）

1. `start-https-server.bat` をダブルクリック
2. コマンドプロンプトが開き、サーバーが起動します
3. ブラウザで `https://localhost:5500` にアクセス

### 方法2: PowerShellスクリプトを使用

1. PowerShellで `.\start-https-server.ps1` を実行
2. ブラウザで `https://localhost:5500` にアクセス

### 方法3: npmコマンドを使用

```bash
npm run https
```

## 初回起動時の注意

初回起動時は、証明書が自動生成されます（数秒かかります）。

## ブラウザでの警告について

自己署名証明書を使用しているため、初回アクセス時にブラウザで警告が表示されます。

### Chrome / Edge の場合

1. 「この接続ではプライバシーが保護されません」という画面が表示されます
2. **「詳細設定」** をクリック
3. **「localhost にアクセスする（安全ではありません）」** をクリック

### Firefox の場合

1. 「警告: 潜在的なセキュリティリスク」という画面が表示されます
2. **「詳細情報」** をクリック
3. **「リスクを承知して続行」** をクリック

## トラブルシューティング

### ERR_SSL_PROTOCOL_ERROR が表示される場合

1. サーバーが正しく起動しているか確認してください
2. コマンドプロンプト/PowerShellでエラーメッセージを確認してください
3. ポート5500が既に使用されている場合は、別のポートに変更してください

### 証明書の生成に失敗する場合

以下のコマンドで証明書を手動生成できます：

**Windows:**
```bash
generate-cert.bat
```

**PowerShell:**
```powershell
.\generate-cert.ps1
```

### ポートが使用されている場合

`https-server.js` の `PORT` 変数を別の番号（例: 5501）に変更してください。
