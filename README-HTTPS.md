# HTTPSサーバーの起動方法

## ⚠️ 重要: 正しいディレクトリで実行してください

HTTPSサーバーを起動するには、**プロジェクトのディレクトリ**（`supabase-data-viewer`フォルダ）で実行する必要があります。

## 起動方法

### 方法1: バッチファイルを使用（最も簡単・推奨）⭐

1. エクスプローラーで `supabase-data-viewer` フォルダを開く
2. `start-https-simple.bat` を**ダブルクリック**

これで自動的に証明書が生成され、サーバーが起動します。

### 方法2: コマンドプロンプトから実行

1. コマンドプロンプトを開く
2. **プロジェクトディレクトリに移動**:
   ```cmd
   cd "C:\Users\yoshida\データ管理\supabase-data-viewer"
   ```
3. サーバーを起動:
   ```cmd
   node https-server.js
   ```

### 方法3: PowerShellから実行

1. PowerShellを開く
2. **プロジェクトディレクトリに移動**:
   ```powershell
   cd "C:\Users\yoshida\データ管理\supabase-data-viewer"
   ```
3. サーバーを起動:
   ```powershell
   node https-server.js
   ```

## サーバーが起動したら

以下のメッセージが表示されます：

```
========================================
✅ HTTPSサーバーが起動しました！
URL: https://localhost:5500
========================================
```

## ブラウザでアクセス

1. ブラウザで `https://localhost:5500` にアクセス
2. 警告が表示された場合：
   - **Chrome/Edge**: 画面下部の「**詳細設定**」をクリック → 「**localhost にアクセスする（安全ではありません）**」をクリック
   - **見つからない場合**: アドレスバーに `thisisunsafe` と入力（表示されませんが入力するとアクセスできます）
   - **Firefox**: 「**詳細情報**」をクリック → 「**リスクを承知して続行**」をクリック

## トラブルシューティング

### エラー: "Cannot find module"

**原因**: 間違ったディレクトリで実行している

**解決方法**: 
- プロジェクトディレクトリ（`supabase-data-viewer`フォルダ）に移動してから実行してください
- または、`start-https-simple.bat` をダブルクリックしてください（自動的に正しいディレクトリで実行されます）

### エラー: "ポート5500は既に使用されています"

**解決方法**: 
- 既に起動しているサーバーを停止してください（Ctrl+C）
- または、`https-server.js` の `PORT` 変数を別の番号に変更してください

### 証明書の生成エラー

**解決方法**:
```cmd
npm install --save-dev selfsigned
```

## 停止方法

サーバーを停止するには、コマンドプロンプト/PowerShellで **Ctrl+C** を押してください。
